# Ken AI Campaign Configuration Reference

This reference covers how AI personalization prompts are configured and executed in Ken AI, extracted from the Campaign Configuration Guide.

---

## AI Personalization Stage Overview

AI personalization is Stage 6 of the Ken AI campaign workflow. Use this stage when:
- Your email sequence includes personalization variables
- You're using data-driven hooks or compliments
- Your value prop needs to be tailored per prospect

If your email sequence is generic and doesn't include variables, skip this stage.

---

## How AI Personalization Works

1. Use `/campaign-strategy` to design email sequence with variables
2. Use `/prompt-writer` to create user prompts (output variables) for each AI variable
3. Backend auto-generates system prompts (Instructions, Rules, Company Context, Email Sequence) from campaign data
4. System automatically provides relevant data to AI
5. AI generates content for each variable per contact

---

## Prompt Types in Ken App

### Context Prompts (`to_output: false`) - Auto-Generated
- Auto-generated by the backend from campaign data and `Constants.*` templates
- Provide background information to AI (Instructions, Rules, Company Context, Email Sequence)
- CSMs do not need to manually create these

### Output Prompts (`to_output: true`)
- Generate content that goes into email
- Becomes a database field/variable
- Example: `{personalized_hook}`, `{pain_point}`, `{social_proof}`

## Available Data Sources

Prompts can only reference data that has been enriched for the campaign. Standard enrichments include:

### LinkedIn Profile
- Full experience history, position descriptions
- Skills, education, certifications
- Useful for role-specific personalization

### LinkedIn Company
- Industry, headcount, headquarters
- Company description, website domain, follower count
- Required for most campaigns (provides domain)

### Website Content
Available page types:
- Home (default)
- About
- Services & Products
- Pricing
- Testimonials, Case Studies
- Team
- Careers

**Cost**: $2.5 per 1,000 pages scraped

**Best Practice**: Homepage alone is often sufficient. Add Services/Products page only if personalization discusses their offerings.

### LinkedIn Posts (Rare)
- Recent posts (last 30 days)
- Only useful for active LinkedIn creators
- Most prospects don't post regularly - avoid unless targeting confirmed posters

### Google Search (Expensive)
- Custom search query with variables: `{COMPANY_NAME} competitors`
- Returns first page of results, optionally scrapes top 3
- Reserve for high-value campaigns where specific intelligence is critical

## AI Rewriting

AI Rewriting is now handled automatically by the backend with sensible defaults. The backend uses `Constants.DefaultRewritingPrompt` as a fallback when no user-defined rewriting prompts exist.

### How It Works
- The `to_rewrite` field on each user prompt controls whether its output gets rewritten
- `to_rewrite=0` - NOT rewritten (use for subject lines, short outputs)
- `to_rewrite=1` - WILL be rewritten (use for First Line, PS Line, body content)
- No manual rewriting prompt configuration is needed

### When NOT to Rewrite
- Subject lines (too risky to rewrite)
- Very simple/predictable prompts (e.g., `{firstName}`)
- Extremely formulaic outputs where consistency matters more than variation

## Best Practices

1. **Limit to 3-5 output variables per email** - More variables = longer processing time
2. **Test prompts with sample data before scaling** - Catch issues early
3. **Verify data sources are enabled** - Prompts can only use enriched data
4. **Provide clear fallback instructions** - Handle missing data gracefully
5. **Match variable names exactly** - Strategy and prompts must use same names

---

## Troubleshooting Poor AI Personalization Quality

### Possible Causes
- Insufficient data enrichment
- Vague AI prompts
- Not using AI Rewriting

### Solutions
1. Review `/prompt-writer` output for specificity
2. Ensure relevant data enrichments are enabled in campaign
3. Add AI Rewriting step
4. Test prompts on sample data before full deployment
5. Provide clearer instructions and examples in prompts
